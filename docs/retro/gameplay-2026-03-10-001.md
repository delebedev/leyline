## Issues Encountered

1. **macOS Dock blocked hand cards.** The dock overlay prevented card interaction at y~530+. Fixed by auto-hiding the dock (`System Events` → `autohide`). This should be a pre-flight check.

2. **`arena drag` didn't work initially.** Cards at y=530 were below the visible area. After hiding the dock, dragging from OCR card-name coordinates worked.

3. **Click-to-play didn't work.** Unity/Arena requires drag to play cards, not click. Clicking a card opens inspection/preview. The `arena drag` command is the correct approach (documented in arena-cli.md).

4. **Debug API returned empty state.** `curl localhost:8090/api/state` showed `turn: 0, phase: null` throughout the game. `api/id-map` returned empty data. This meant `arena play <name>` couldn't find cards. OCR + drag was the only viable method.

5. **Combat auto-skipped frequently.** When only summoning-sick creatures could attack, the game jumped from "Next To Combat" straight to "End Turn" — no "Choose attackers" prompt. This is expected Arena behavior but makes attack tracking harder.

6. **"Pass To Attackers" stuck loop.** During Sparky's turn, the "Pass To Attackers" button sometimes required 4-6 clicks before the game advanced. Unclear if this was a timing issue or the server holding priority.

7. **Damage assignment screen.** Sparky blocked with turtles frequently, triggering the "Assign Damage" modal. Handled with "Auto Allocate" + "Done" — worked reliably.

8. **Card hand positions shifted** after each card played, making hardcoded x-coords unreliable. Using OCR card-name text as drag origin worked but card names aren't always detected at small sizes.

## Suggestions for Improving Automation

1. **Pre-flight dock check.** Auto-hide the dock before gameplay starts, restore after.

2. **Fix debug API integration.** The empty id-map/state means `arena play` and `arena board` are useless. This is the biggest blocker — with working API data, `arena play <name>` would eliminate all drag-coordinate guesswork.

3. **Land play via enlarged cards.** When clicking the hand area where lands are, Arena shows enlarged land options above the hand. Clicking the enlarged card plays it directly. This was the most reliable land-play method discovered.

4. **Claws Out was never cast.** At 3 mana it was always competing with better plays (Banishing Light, Good-Fortune Unicorn). A smarter casting priority would help: land → cheap creatures → removal → combat tricks.

5. **Attacking needs explicit stop.** The game auto-skips combat when only summoning-sick creatures are available. Setting a manual stop at "Beginning of Combat" would ensure we always get the attackers prompt.

6. **Turn tracking without API.** Without the debug API, turn count had to be inferred from context. A simple counter in the automation loop would help decide when to concede.
